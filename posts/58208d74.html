计算机系统结构基础知识（1-28）

计算机系统的设计

计算机系统设计的定量原理

Amdahl定理

加速比：改进前总执行时间（T0） / 改进后总执行时间（Tn）

Tn = T0（1- Fe + Fe / Se ）

Fe(功能所占的时间比例) = 可改进时间/总改进时间

Sn（处理速度的加快倍数） = 改进前所需执行时间 / 改进后所执行时间

Sn = 1 / [ (1 - Fe ) + Fe/ Se ]

CPU性能公式

CPU时间 = 执行程序所需时钟周期 / 时钟周期时间

CPI：每条指令的平均时钟周期 = 执行指令所需的时钟周期数 / 指令个数

IC：指令条数

CPU时间 = CPI * IC * 一个时钟周期的时间

局部性原理

空间局部性：访问存储器的地址不是随机的，而是簇拥的。

时间局部性：访问的信息很可能是最近刚访问的

计算机系统结构的发展

冯诺依曼结构及其改进

冯诺依曼：运算器，控制器，存储器，输入，输出。以运算器为中心，指令和数据对等，内存按地址访问，输入和输出是串行的。

指令系统的设计(30-51)

指令系统结构的分类

指令系统结构的分类按照存储单元分类分为：堆栈型结构，累加器型结构，通用寄存器型结构（分为寄存器-存储器型结构，寄存器-寄存器型结构）。

堆栈型结构

操作的数据都是隐式的，只能操作栈顶和次栈顶的数据，即只能用push和pop进行操作。

push X

push Y

add

pop Z

累加器型结构

一个操作数是隐式的，另一个是显式的（存储单元）。

load X

add Y

store Z

通用寄存器型

操作数都是显式给出的，要么来自于寄存器，要么来自于存储器，结果写入通用寄存器组。

RM:

load R1, X

add R1, Y

store R1, Z

RR:

load R1, X

load R2, Y

add R3, R1, R2

store R3, Z

目前都在用通用寄存器组结构，因为：

1.访问速度比存储器快

2.能更有效的分配寄存器和使用寄存器

3.寄存器可以存放变量

寻址方式

PC相对寻址：以程序计数器PC作为参考，计算偏移量得到物理地址的方式叫做相对寻址

立即数寻址和偏移寻址的方式，使用频度最高。

如何表示寻址方式？

①隐含在操作码当中

②在指令中设置专门的字段声明寻址方式

如何存储不同宽度的信息？

信息分为了四种：字节，半字，单字，双字。要求信息开始的地址必须是一个存储字的整数倍，也就是整数边界的概念

流水线技术(53-90)

流水线的基本概念

流水线：将一个重复的过程分成小的部分，多个处理过程在时间上错开，比如同一时刻可处理过程1和过程3，但不可能出现同一时刻处理过程1和过程1

段：流水线的一个过程

流水线的深度：流水线的段数

通过时间：第一个任务完成的时间段

排空时间：最后一个任务完成的时间段

流水寄存器：流水线每一个段的后面都要有一个缓冲寄存器，在相邻的两段之间传送数据，以保证提供后面要用到的信息，并把各段的处理工作相互隔离

流水线的分类

按计算机系统的等级划分
部件级流水线

按处理机的部件分段

处理机级流水线

按指令的执行过程分段

系统级流水线

多个处理机串行连接

按完成的功能划分
单功能流水线

只完成一种功能

多功能流水线

完成多种功能

各段之间的连接方式
静态流水线

在同一时间内，多功能流水线中的各段只能按同一种功能的连接方式工作。

对于静态流水线来说，只有当输入的是一串相同的运算任务时，流水的效率才能得到充分的发挥。

动态流水线

在同一时间内，多功能流水线中的各段可以按照不同的方式连接，同时执行多种功能。

优缺点：灵活，但是控制复杂

是否有反馈回路
线性流水线

流水线的各段串行连接，没有反馈回路。数据通过流水线中的各段时，每一个段最多只流过一次。

非线性流水线

流水线中除了有串行的连接外，还有反馈回路。

任务流入和流出的顺序是否相同
顺序流水线

流水线输出端任务流出的顺序与输入端任务流入的顺序完全相同。每一个任务在流水线的各段中是一个跟着一个顺序流动的

乱序流水线

流水线输出端任务流出的顺序与输入端任务流入的顺序可以不同，允许后进入流水线的任务先完成（从输出端流出）

流水线的性能指标

吞吐率

TP = n / Tk

n:任务数，Tk：完成n个需要的时间

各段时间相等的流水线

Tk = （k + n - 1 ）* ▲t

k为段数，n为任务数。

TP = n / Tk

TPmax = 1 / ▲t

各段时间不相等的流水线

加速比

完成同样一批任务，不使用流水线所用的时间与使用流水线所用的时间之比。

不使用流水线（即顺序执行）所用的时间为Ts，使用流水线后所用的时间为Tk

各段时间相等的流水线

S = nk / (k + n - 1)

Smax= k

各段时间不相等的流水线

流水线的效率

流水线中的设备实际使用时间与整个运行时间的比值，即流水线设备的利用率

各段时间相等

最高效率为1

流水线设计中的若干问题

瓶颈问题

当流水线各段不均匀时，机器的时钟周期取决于瓶颈段的延迟时间.在设计流水线时，要尽可能使各段时间相等

流水线的额外开销
流水寄存器延迟

流水寄存器需要建立时间和传输延迟.

建立时间：在触发写操作的时钟信号到达之前，寄存器输入必须保持稳定的时间

传输延迟：时钟信号到达后到寄存器输出可用的时间

时钟偏移开销

流水线中，时钟到达各流水寄存器的最大差值时间

非线性流水线的调度

在非线性流水线中，存在反馈回路，当一个任务在流水线中流过时，可能要多次经过某些段.

流水线调度要解决的问题：应按什么样的时间间隔向流水线输入新任务，才能既不发生功能段使用冲突，又能使流水线有较高的吞吐率和效率？

单功能非线性流水线的最优调度

启动距离：向一条非线性流水线的输入端连续输入两个任务之间的时间间隔

禁用启动距离：会引起非线性流水线功能段使用冲突的启动距离

预约表

横向（向右）：时间（一般用时钟周期表示）

纵向（向下）：流水线的段

如果在第n个时钟周期使用第k段，则在第k行和第n列的交叉处的格子里有一个√；如果在第k行和第n列的交叉处的格子里有一个√，则表示在第n个时钟周期要使用第k段。

禁止表

一个由禁用启动距离构成的集合.

具体方法：对于预约表的每一行的任何一对√，用它们所在的列号相减（大的减小的），列出各种可能的差值，然后删除相同的，剩下的就是禁止表的元素

ci=1 ：不允许间隔i个时钟周期后送入后续任务

调度方案：根据流水线状态图，由初始状态出发，任何一个闭合回路即为一种调度方案.

流水线的相关与冲突

经典的5段RISC流水线

经典的5段RISC流水线执行过程分为以下5个周期：

1.取指令周期.以程序计数器PC中的内容作为地址，从存储器中取出指令并放入指令寄存器IR；同时PC值加4（假设每条指令占4个字节），指向顺序的下一条指令。

2.指令译码/读寄存器周期.对指令进行译码，并用IR中的寄存器地址去访问通用寄存器组，读出所需的操作数

3.执行/有效地址计算周期.

4.存储器访问／分支完成周期.该周期处理的指令只有load、store和分支指令

5.写回周期.ALU运算指令和load指令在这个周期把结果数据写入通用寄存器组

流水线冲突

对于具体的流水线来说，它使得指令流中的下一条指令不能在指定的时钟周期执行.

带来的问题：

结构冲突

因硬件资源满足不了指令重叠执行的要求而发生的冲突.

原因

1.功能部件不是完全流水

2.资源没有充分重复设置

举例：访存冲突

些流水线处理机只有一个存储器，将数据和指令放在一起，访存指令会导致访存冲突.

解决办法1：插入暂停周期（“流水线气泡”或“气泡”）

解决方法2：设置相互独立的指令存储器和数据存储器或设置相互独立的指令Cache和数据Cache。

数据冲突

当指令在流水线中重叠执行时，因需要用到前面指令的执行结果而发生的冲突.

举例

解决方法：通过旁路技术减少数据冲突引起的停顿(并不是所有的数据冲突都可以用旁路技术来解决)

在计算结果尚未出来之前，后面等待使用该结果的指令并不真正立即需要该计算结果，如果能够将该计算结果从其产生的地方直接送到其它指令需要它的地方，那么就可以避免停顿

控制冲突

流水线遇到分支指令和其它会改变PC值的指令所引起的冲突.

处理分支指令最简单的方法：“冻结”或者“排空”流水线.

一旦在译码段ID检测到分支指令，就暂停其后所有指令的执行，直到分支指令到达MEM段。

分支延迟：由分支指令引起的延迟.

减少分支延迟：

1.在流水线中尽早判断出分支转移是否成功

2.尽早计算出分支目标地址

减少分支延迟的方法

1.预测分支失败

1.允许分支指令后的指令继续在流水线中流动，就像啥都没发生

2.确定分支失败，将分支指令看作是一条普通指令，流水线正常流动

3.确定分支成功，流水线就把在分支指令之后取出的所有指令转化为空操作，并按分支目地重新取指令执行

2.预测分支成功

假设分支转移成功，并从分支目标地址处取指令执行

3.延迟分支

从逻辑上“延长”分支指令的执行时间。把延迟分支看成是由原来的分支指令和若干个延迟槽构成，不管分支是否成功，都要按顺序执行延迟槽中的指令

横向处理
